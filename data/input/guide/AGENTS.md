# Guide Content Instructions

## Scope
- This folder contains active numbered guide articles only.
- Keep durable guide content here; do not keep drafts, archives, or temporary split files in this directory.

## File Rules
- Use stable numbered filenames such as `23_basic_prepositions.md`.
- Do not renumber or rename guide files unless the task explicitly requires it.
- Use UTF-8.
- Start each file with a Markdown heading. The first heading is the displayed title.
- Avoid empty files.

## Writing Style
- Write clear, natural Ukrainian for a learner using the app as a reference.
- Treat the guide as a lookup module, not a lesson script.
- Prefer short sections, short paragraphs, and simple bullet lists.
- Lead with the rule, contrast, or model first; then show short examples.
- Keep Hebrew, transliteration, and Ukrainian glosses consistent inside one file.
- Explain grammar terms in everyday language if you use them.
- Write like an experienced textbook author and teacher, not like a template or generated summary.
- Aim for calm, confident, human prose that is easy to read and easy to trust.
- Prefer connected explanation and smooth transitions over chopped, mechanical sentence blocks.

## Voice And Tone
- Sound professional, patient, and literate.
- Favor clarity, rhythm, and natural phrasing over "smart-sounding" wording.
- Let the text feel authored: concise, but not robotic; polished, but not theatrical.
- When two phrasings are both correct, choose the one a good teacher would naturally say to a student.

## Avoid Slop
- Do not sound like a generic AI explainer, translation engine, or note dump.
- Avoid stiff meta-phrases like "this topic helps", "this model is useful", "a simple rule", or "in practice it is important" unless they truly add meaning.
- Avoid repeating the same sentence openings, the same framing formulas, or the same abstract nouns across nearby paragraphs.
- Avoid inflated textbook bureaucracy such as "this section functions as", "this topic is important for", or "within this construction".
- Avoid over-compressing ideas into slogan-like lines; explanations should flow like human prose.
- Avoid decorative cleverness, marketing tone, exaggerated friendliness, and direct calques if simpler Ukrainian exists.

## Structure
- Keep one central topic per file.
- If the topic is broad, split it into tight subsections rather than mixing several themes.
- A typical article should include:
  - short intro
  - main model or contrast
  - forms / variants if needed
  - short examples
  - common mistakes where useful
  - related topics

## Do Not Add
- Full lesson flow
- drills or homework blocks
- spaced repetition advice
- mini-dialogue sections unless explicitly requested
- HTML, tables, or complex Markdown

## Editorial Priorities
- Favor correctness over flourish.
- Favor natural phrasing over textbook prose.
- Favor short, retrievable explanations over long paraphrases.
- Call out common Ukrainian-to-Hebrew interference when it prevents mistakes.
- Prefer "written by a seasoned teacher" over "efficiently structured content".
- If a paragraph sounds generated, flattened, or interchangeable, rewrite it instead of lightly polishing it.

## Notes
- Helper files like this `AGENTS.md` are not lessons.
- Stay aligned with neighboring guide files in tone and difficulty.
