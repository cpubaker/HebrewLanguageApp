# План розширення beginner reading

## Ціль

Розширити блок `data/input/reading/beginner/` з 11 до 31 уроку без зміни існуючих файлів. Нові тексти мають:

- залишатися дуже простими для новачків;
- використовувати нікуд у всьому івриті;
- тримати формат: заголовок, основний текст, `## Основні слова`, за потреби `## Базові дієслова`;
- рухатися від окремих побутових сцен до коротких міні-історій.

## Робочі принципи

- Один текст = одна ситуація.
- У кожному новому тексті 6-12 коротких речень.
- Приблизно 70-80% слів мають бути вже знайомими з попередніх уроків.
- Нові дієслова вводимо невеликими порціями і повторюємо в наступних текстах.
- Кожен пакет уроків має спиратися на сусідні теми з `guide`.

## Порядок написання

1. Спершу написати уроки `12-16` як тестовий пакет.
2. Перевірити стиль, середню довжину тексту і щільність нової лексики.
3. Потім написати `17-21`, не збільшуючи різко складність.
4. Після цього рухатися пакетами `22-26`, `27-30`, `31-36`.

## Внутрішній шаблон для кожного нового файлу

```md
# Назва

Короткий beginner-текст із 2-4 абзаців.

## Основні слова

- ...

## Базові дієслова

- ...
```

## Прив'язка до guide

- `12-16`: артикль, рід/число, `יֵשׁ`, базові прийменники.
- `17-21`: питання, модальні слова, числа, ввічливі прохання.
- `22-26`: заперечення, накази, часові маркери, порядок дій.
- `27-30`: прикметники, стан, прості зв'язки `אֲבָל`, `וְלָכֵן`.
- `31-36`: міні-історії з кількох сцен як місток до наступного рівня.

## Definition of done для кожного пакета

- У всьому івриті є нікуд.
- Заголовок і структура відповідають `reading/AGENTS.md`.
- Глосарій покриває ключові нові слова з тексту.
- Дієслова в кінці справді використані в тексті або прямо потрібні для його розуміння.
- Після пакета запускаємо `python -m unittest discover -s tests -v`.
